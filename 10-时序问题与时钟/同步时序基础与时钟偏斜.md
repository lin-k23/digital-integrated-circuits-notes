# 同步时序基础与时钟偏斜

## 一句话理解

边沿触发同步系统的核心约束有两类：setup 约束决定最高频率，hold 约束决定是否会发生 race。clock skew 会改变发射寄存器和接收寄存器之间的有效时间：正 skew 可能放宽 setup，却压缩 hold；负 skew 通常牺牲性能，但降低 race 风险。

## 基本同步路径

考虑典型同步路径：

```text
R1 -> combinational logic -> R2
```

其中：

- $R_1$ 是 launching register。
- $R_2$ 是 capturing register。
- $t_{c-q}$：$R_1$ 的最大 clock-to-Q 延迟。
- $t_{c-q,cd}$：$R_1$ 的最小 clock-to-Q / contamination delay。
- $t_{logic}$：组合逻辑最大传播延迟。
- $t_{logic,cd}$：组合逻辑最小污染延迟。
- $t_{su}$：$R_2$ 的 setup time。
- $t_{hold}$：$R_2$ 的 hold time。

## 理想 clock 下的 setup 约束

若 $R_1$ 与 $R_2$ 的 clock 边沿完全同时到达，则最小时钟周期需要满足：

$$
T \ge t_{c-q}+t_{logic}+t_{su}
$$

直觉：

```text
一拍时间 >= 发射寄存器出数时间 + 组合逻辑最慢计算时间 + 接收寄存器建立时间
```

## 理想 clock 下的 hold 约束

同一条路径还必须满足 hold：

$$
t_{c-q,cd}+t_{logic,cd} \ge t_{hold}
$$

直觉：

```text
新数据不能太快到达接收寄存器输入，否则会破坏接收寄存器对旧数据的 hold 要求
```

## clock skew 定义

clock skew 是 clock transition 到达芯片不同位置的空间差异。

对路径 $R_1\to R_2$，本笔记统一定义：

$$
\delta=t_{CLK2}-t_{CLK1}
$$

其中：

- $t_{CLK1}$：clock 到达发射寄存器 $R_1$ 的时间。
- $t_{CLK2}$：clock 到达接收寄存器 $R_2$ 的时间。
- $\delta>0$：接收端 clock 比发射端晚到，称为 positive skew。
- $\delta<0$：接收端 clock 比发射端早到，称为 negative skew。

> 注意：后续所有 skew 公式都使用这个符号约定。

## 有 skew 时的 setup 约束

接收端 clock 如果晚到 $\delta$，数据可用时间增加 $\delta$。因此：

$$
T+\delta \ge t_{c-q}+t_{logic}+t_{su}
$$

等价写法：

$$
T \ge t_{c-q}+t_{logic}+t_{su}-\delta
$$

含义：

- $\delta>0$：setup 约束放宽，看似可以提高频率。
- $\delta<0$：setup 约束变紧，最小时钟周期增加。

## 有 skew 时的 hold 约束

正 skew 会让接收端下一次采样窗口“晚到”，但 hold 检查关注的是同一拍旧数据需要保持多久。若新数据太快从 $R_1$ 传播到 $R_2$，会产生 race。

约束为：

$$
\delta+t_{hold}\le t_{c-q,cd}+t_{logic,cd}
$$

等价写法：

$$
\delta\le t_{c-q,cd}+t_{logic,cd}-t_{hold}
$$

含义：

- 正 skew 不能无限增加。
- 如果 $\delta$ 太大，新数据可能在接收寄存器完成 hold 之前到达。
- 这种错误与 clock period 无关，降低频率不能修复。

## 正 skew

$\delta>0$ 对应 clock 与数据流方向相同：

```text
CLK reaches R1 first, then R2
```

优点：

- 增加 R1 到 R2 的可用计算时间。
- 可能提高吞吐率。

风险：

- 增加 hold/race 风险。
- 超过 hold 约束后，电路功能错误，与频率无关。

## 负 skew

$\delta<0$ 对应接收端 clock 早于发射端：

```text
CLK reaches R2 before R1
```

优点：

- 对 hold/race 更安全。
- 新数据不会在同一接收边沿前过早到达。

代价：

- 可用计算时间减少。
- setup 约束更紧，最高频率下降。

## false path 提醒

教材例 10.1 强调：组合逻辑最大延迟不能只靠“把最多 gate 延迟相加”得到。

原因：

- 某些路径在逻辑上永远不会被激活，叫 false path。
- critical path 依赖电路拓扑和 data dependency。
- timing 分析必须区分真实可达路径和伪路径。

## 设计直觉

- setup failure 通常是“太慢”。
- hold failure 通常是“太快”。
- 正 skew 对 setup 有利，对 hold 不利。
- 负 skew 对 hold 有利，对 setup 不利。
- hold/race 错误不能靠降低 clock frequency 修复。
- 低 skew clock network 是高性能同步设计的核心基础。

## 易错点

- 不要只看 setup 公式而忽略 hold。
- 不要把正 skew 简单理解成“总是好事”。
- 不要用降频修复 hold violation；应增加最小延迟、调整 clock skew 或修改 clock tree。
- 不要把 longest topological path 直接当作 worst-case propagation delay。
- 不要在不同资料之间混用 skew 符号约定。
