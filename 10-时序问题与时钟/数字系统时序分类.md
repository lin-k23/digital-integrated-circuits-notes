# 数字系统时序分类

## 一句话理解

数字系统里的信号可以按它和本地 clock 的关系分类：同步信号频率和相位都已知；mesochronous 信号频率相同但相位未知；plesiochronous 信号频率近似相同但会缓慢漂移；asynchronous 信号不受本地 clock 约束，需要同步器或握手机制处理。

## 分类维度

教材按照信号与本地 clock 的关系分类：

| 类型 | 频率关系 | 相位关系 | 常见处理方式 |
|---|---|---|---|
| synchronous | 完全相同 | 固定且已知 | 直接采样 |
| mesochronous | 完全相同 | 未知固定偏移 | 相位调整 / variable delay / mesochronous synchronizer |
| plesiochronous | 名义相同但略有差异 | 相位随时间漂移 | timing recovery + FIFO |
| asynchronous | 任意变化 | 不从属于本地 clock | synchronizer 或 self-timed handshaking |

## synchronous interconnect

同步信号满足：

```text
same frequency + known fixed phase offset
```

因此接收寄存器可以在已知时间窗口内采样。

直觉：

- 数据流跟着系统 clock lockstep 前进。
- 只要满足 setup/hold，采样不需要额外同步机制。
- 最直接、最常见、最容易验证。

典型结构：

```text
R1 -> combinational logic -> R2
```

其中 $R_1$ 的输出经过组合逻辑，在一个 clock 周期内到达 $R_2$ 并满足 setup。

## mesochronous interconnect

mesochronous 信号满足：

```text
same frequency + unknown phase offset
```

典型场景：

- 两个模块使用同频 clock。
- 由于 clock distribution 或互连延迟不同，相位未知。
- 发送端数据相对接收端 clock 不一定落在安全采样窗口。

处理方式：

- 使用 variable delay line。
- 用 phase detector / control 调整数据相位。
- 对齐后再由接收寄存器采样。

## plesiochronous interconnect

plesiochronous 信号满足：

```text
nominally same frequency + slightly different frequency
```

也就是说两个 clock 频率名义上一样，但来自不同 oscillator，实际频率有微小差异，因此相位会随时间漂移。

典型场景：

- 长距离通信。
- 分布式系统。
- 两个模块有独立晶振。

处理方式：

- timing recovery。
- FIFO buffering。
- 频率偏差累积过大时需要处理 overflow / underflow。

直觉：

- mesochronous 是“相位未知但不漂移”。
- plesiochronous 是“相位会慢慢漂移”。

## asynchronous interconnect

异步信号满足：

```text
can transition at arbitrary time
```

它不从属于本地 clock，因此不能假设会在某个安全窗口内变化。

常见处理方式有两类：

1. 放到同步系统里处理：用 synchronizer 采样，降低 metastability 风险。
2. 采用 self-timed / asynchronous design：不用全局 clock，用 Req/Ack handshaking 规定事件顺序。

自定时结构中，模块之间通过握手信号控制：

```text
Req -> Data valid -> Ack
```

## 四类时序关系的记忆方式

```text
synchronous      = 同频 + 相位已知
mesochronous     = 同频 + 相位未知
plesiochronous   = 近同频 + 相位漂移
asynchronous     = 任意时刻变化
```

## 设计直觉

- 越接近 synchronous，越容易直接采样。
- 相位未知但频率相同，可以通过相位调整解决。
- 频率略不同，必须用 FIFO 或协议吸收速率差。
- 完全异步输入不能直接当同步信号使用，否则可能触发 metastability。
- 跨 clock domain 的难度取决于频率关系、相位关系和数据吞吐要求。

## 易错点

- 不要把 mesochronous 和 plesiochronous 混淆。
- 不要以为同频就能直接采样；相位未知仍然需要同步/对齐。
- 不要把异步信号直接送入多个同步寄存器，会造成状态不一致。
- FIFO 解决的是速率/相位漂移问题，不是单点 metastability 的万能解。
