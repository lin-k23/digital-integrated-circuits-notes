# 组合逻辑章节总览

## 一句话理解

组合逻辑（combinational logic）的输出只由当前输入决定；第 6 章的目标是把第 5 章的 CMOS 反相器设计方法推广到 NAND、NOR、XOR、复杂门以及多种 CMOS 逻辑风格。

- 对应小节：6.1 Introduction

## 组合逻辑与时序逻辑

组合逻辑的特点是：

```text
当前输出 = 当前输入的布尔函数
```

只要瞬态过程结束，输出不依赖过去历史。典型组合逻辑包括：

- NAND / NOR / XOR 等逻辑门。
- 加法器中的组合路径。
- 多路选择器（multiplexer）。
- 编码器 / 译码器。

时序逻辑（sequential logic）则不同：输出不仅依赖当前输入，还依赖过去状态。例如寄存器、计数器、存储器等。

## 本章关注的设计指标

和 [[../05-CMOS反相器/README|CMOS反相器]] 一样，组合逻辑门也要按多个指标评价：

| 指标 | 第 6 章关注点 |
|---|---|
| 面积（area） | 晶体管数量、逻辑风格、版图复杂度 |
| 速度（speed） | 传播延迟、串联堆栈、负载、电气努力 |
| 能量（energy） | 负载电容、切换活动、短路和泄漏 |
| 鲁棒性（robustness） | 噪声容限、满摆幅、低阻输出、工艺偏差 |
| 可靠性（reliability） | 动态节点泄漏、串扰、电源缩放下的裕量 |

不同应用侧重点不同：高性能处理器更重视速度，电池供电系统更重视能量，通用标准单元库则重视自动化、鲁棒性和可验证性。

## 本章逻辑风格

第 6 章讨论多种 CMOS 逻辑实现方式：

1. 静态互补 CMOS（static complementary CMOS）
2. 比例逻辑（ratioed logic）
3. 传输管逻辑（pass-transistor logic）
4. 动态 CMOS 逻辑（dynamic CMOS logic）
5. domino / np-CMOS 等动态逻辑扩展

本轮优先整理最重要、最常用的静态互补 CMOS。

## 为什么从静态 CMOS 开始

静态互补 CMOS 是工业中最常用的组合逻辑风格，因为它：

- 满摆幅输出。
- 无比例逻辑，逻辑电平不依赖尺寸比。
- 理想稳态无静态功耗。
- 噪声容限好。
- 适合自动综合和标准单元库。

缺点是复杂门可能需要较多晶体管，并且高 fan-in 门会因为串联堆栈变慢。

## 和前后章节的连接

- 第 5 章 [[../05-CMOS反相器/README|CMOS反相器]] 是单输入静态 CMOS 门。
- 第 6 章把这个结构推广为多输入 PUN / PDN 网络。

## 设计直觉

- 先用静态 CMOS 获得正确、鲁棒的实现。
- 再根据面积、速度或功耗需求考虑其他逻辑风格。
- 高 fan-in、长串联堆栈和大电容负载是组合逻辑延迟的主要来源。
- 逻辑功能、晶体管拓扑、版图寄生和输入活动模式共同决定门的表现。

## 易错点

- 不要把组合逻辑和时序逻辑混淆；组合逻辑没有有意反馈和状态保持。
- 不要认为所有布尔函数都适合用单级复杂门实现；复杂门可能慢且面积大。
- 不要只看晶体管数量，噪声容限、功耗和可验证性也很重要。
- 不同逻辑风格不是越高级越好，互补 CMOS 仍然是默认首选。
