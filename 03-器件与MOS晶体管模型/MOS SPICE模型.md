# MOS SPICE 模型

## 一句话理解

MOS SPICE 模型是把短沟道 MOS 的电流、电容、温度、噪声和版图寄生效应用大量参数封装起来，让仿真器能在不同尺寸、偏置和工艺角下计算器件行为。

## 为什么 MOS SPICE 模型复杂

前面的 [[MOS工作区与电流模型]] 适合手工理解，但它故意忽略了很多效应，例如：

- 短沟道效应。
- 速度饱和。
- 沟道长度调制。
- 迁移率退化。
- DIBL 和穿通。
- 源/漏串联电阻。
- 非线性电容。
- 温度依赖。
- 噪声。

这些效应对实际仿真很重要，因此 SPICE 模型通常远比手工模型复杂。

## 模型精度与计算代价

SPICE 模型的核心权衡是：

```text
模型越精确 → 参数越多 → 计算越慢 → 使用和验证越困难
```

设计直觉：

- 手算模型用于理解趋势。
- SPICE 模型用于验证电路。
- 工艺厂模型用于签核，但也不是“真实世界的全部”。
- 仿真精度不可能超过模型和版图寄生信息本身的精度。

## BSIM 类模型

现代短沟道 CMOS 常用 BSIM 类模型。教材中提到的 BSIM3v3 / LEVEL-49 属于此类。

模型参数大致分成几类：

| 参数类别 | 作用 |
|---|---|
| 控制参数 | 选择模型级别、迁移率模型、电容模型和噪声模型 |
| DC 参数 | 阈值、电流、短沟道效应等直流特性 |
| AC 与电容参数 | 小信号电容、非线性电容、频率响应 |
| $dW$ 与 $dL$ | 从版图尺寸推导有效沟道宽度和长度 |
| 工艺参数 | 氧化层厚度、掺杂浓度、结深等 |
| 温度参数 | 标称温度和温度系数 |
| bin 参数 | 模型适用的尺寸范围 |
| 噪声参数 | 闪烁噪声等 |

## binning：为什么同一工艺有多组模型

同一工艺下，不同尺寸的 MOS 管可能不能用同一组模型参数精确描述。SPICE 模型常按照尺寸范围划分 bin：

- $L_{min}$ 到 $L_{max}$。
- $W_{min}$ 到 $W_{max}$。

仿真器或用户根据器件尺寸选择合适的 bin。

设计直觉：不要把最小尺寸器件拟合出来的模型，随意外推到很长或很宽的器件上。

## 晶体管实例化参数

在 SPICE 网表中，单个 MOS 管除了指定模型名，还应尽量提供版图相关参数：

| 物理意义 | 常见 SPICE 名称 | 说明 |
|---|---|---|
| 沟道长度 | `L` | 版图绘制长度或模型使用长度 |
| 沟道宽度 | `W` | 版图绘制宽度或模型使用宽度 |
| 源区面积 | `AS` | 用于计算源扩散电容 |
| 漏区面积 | `AD` | 用于计算漏扩散电容 |
| 源区周长 | `PS` | 用于计算源侧墙电容 |
| 漏区周长 | `PD` | 用于计算漏侧墙电容 |
| 源扩散方块数 | `NRS` | 乘以薄层电阻，估算源串联电阻 |
| 漏扩散方块数 | `NRD` | 乘以薄层电阻，估算漏串联电阻 |

> 需核对：表格根据 `ch3_en.pdf` Table 3.7 摘要整理，中文扫描版表格建议后续再核对一次。

## 为什么面积和周长很重要

[[MOS寄生电容]] 中的源/漏扩散电容依赖面积和周长：

$$
C_{diff}=C_jA+C_{jsw}P
$$

如果网表没有准确给出 `AS`、`AD`、`PS`、`PD`，仿真器可能使用默认值，导致瞬态仿真中的节点电容严重错误。

同样，源/漏串联电阻依赖 `NRS` 和 `NRD`：

$$
R_{S,D}\approx R_\square NRS/NRD
$$

这会影响导通电流和延迟。

## CMOS 反相器网表示例的含义

教材给出一个 CMOS 反相器的 SPICE 示例：

```spice
M1 nvout nvin 0 0 nmos.1 W=0.375U L=0.25U
+AD=0.24P PD=1.625U AS=0.24P PS=1.625U NRS=1 NRD=1
M2 nvout nvin nvdd nvdd pmos.1 W=1.125U L=0.25U
+AD=0.7P PD=2.375U AS=0.7P PS=2.375U NRS=0.33 NRD=0.33
.lib 'c:\Design\Models\cmos025.l'
```

读法：

- `M1` 是 NMOS，端口顺序通常是 drain, gate, source, body。
- `M2` 是 PMOS，体端接 `nvdd`。
- PMOS 宽度是 NMOS 的 3 倍，用于降低 PMOS 较大的等效电阻。
- PMOS 面积和周长也更大，因此寄生扩散电容更大。
- `.lib` 指向模型库文件。

## 使用 SPICE 的易错点

- 没有给 `AD/AS/PD/PS`，瞬态电容可能严重不准。
- 没有给 `NRS/NRD`，源/漏串联电阻可能被忽略。
- 只用 typical 模型，不做 corner 分析，无法保证最坏情况。
- 对仿真结果过度精确化，例如优化到皮秒或微伏级，通常没有实际意义。
- 把工艺厂模型参数当成物理常数，而不是经验拟合参数。

## 设计直觉

SPICE 不是替代理解的工具，而是验证理解的工具。好的使用方式是：先用手工模型预测趋势，再用 SPICE 检查二阶效应和真实工艺参数。
