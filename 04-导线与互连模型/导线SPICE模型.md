# 导线 SPICE 模型

## 一句话理解

导线 SPICE 模型用于在电路仿真中表示互连线的分布电阻、电容、电感和传输线效应；手工分析给出趋势，SPICE 模型用于检查真实互连寄生对波形和延迟的影响。

## 为什么需要 SPICE 导线模型

前面的 [[导线电路模型]] 已经给出了手工分析模型：

- 集总电容。
- 集总 RC。
- Elmore 延迟。
- 分布 RC 线。
- 传输线。

但真实版图中的互连可能是多段、多分支、多层、带线间耦合和复杂负载的网络。此时只靠手算很难准确预测波形，需要版图寄生提取和 SPICE 仿真。

## 分布 rc 线在 SPICE 中的建模

由于分布 RC 线在现代设计中非常常见，很多 SPICE 仿真器内建了分布 RC 线模型。例如 Berkeley SPICE3 支持 uniform distributed RC line，即 URC 模型。

URC 的直觉是：

```text
连续分布 rc 线 → 内部自动拆成若干 RC 小段 → 用节点网络近似求解
```

常用参数包括：

- 线长 $L$。
- 单位长度电阻 `RPERL`。
- 单位长度电容 `CPERL`。
- 分段数量 `N`。

教材给出的 SPICE3 形式类似：

```spice
U1 N1=1 N2=2 N3=0 URCMOD L=50m N=6
.MODEL URCMOD URC(RPERL=75K CPERL=100pF)
```

其中：

- `N1`、`N2` 是导线两端节点。
- `N3` 是电容参考节点，通常接地。
- `RPERL` 表示每米电阻。
- `CPERL` 表示每米电容。

## 用 RC 梯形网络近似分布线

如果仿真器没有内建分布 RC 线模型，或者内建模型太慢，可以手动用有限段 RC 网络近似。

常见近似包括：

- 一段 $\pi$ 型模型。
- 两段 $\pi_2$ 模型。
- 三段 $\pi_3$ 模型。

段数越多，精度越高，但节点数和仿真成本也越高。

> 图占位：图 4.26 分布 RC 线的 SPICE 近似模型，待后续补图。

设计直觉：

- 一级 RC 近似适合快速估算。
- $\pi_2$ / $\pi_3$ 网络适合更准确的瞬态仿真。
- 段数过多会显著拖慢仿真，不一定带来有意义的设计收益。

## 传输线模型在 SPICE 中的建模

SPICE 支持无损传输线（lossless transmission line）模型。传输线通常由以下参数指定：

- 特性阻抗 $Z_0$。
- 传输延迟 `TD`，也就是 time of flight。
- 或者用频率 `F` 和归一化电长度 `NL` 指定。

二者关系可写为：

$$
NL=F\cdot TD
$$

其中 $NL$ 表示在频率 $F$ 下，传输线长度相对于波长的归一化电长度。

> 需核对：该式按 `ch4_en.pdf` Eq. 4.40 整理，中文扫描版中的符号需后续核对。

## 有损传输线的处理

教材指出，标准 SPICE 中通常没有直接的有损传输线模型。若需要考虑损耗，可以把长传输线分成多段：

```text
短无损传输线段 + 小串联电阻 + 短无损传输线段 + 小串联电阻 + ...
```

这样可以近似导线损耗，但要注意：

- 精度仍有限。
- 段数增加会显著增加仿真时间。
- SPICE 为传输线选择的时间步长通常受 `TD` 限制。
- 过短的小段可能迫使仿真器使用过小时间步，反而比晶体管仿真需求更严格。

## 版图提取和导线模型的关系

真实设计中，导线模型通常不是手工写出来的，而是由版图寄生提取工具生成：

1. 版图给出导线的层、宽度、长度、间距和相邻环境。
2. 提取工具计算寄生 $R$、$C$，必要时计算 $L$ 和耦合项。
3. 生成 SPICE 网表或 SPEF/DSPF 等寄生文件。
4. 仿真器使用这些寄生参数进行时序、噪声和功耗分析。

复习时不必掌握所有文件格式，但要理解：

> 后仿真精度取决于版图寄生提取是否准确，而不仅取决于晶体管模型。

## 模型选择建议

| 场景 | 推荐模型 |
|---|---|
| 很短局部线 | 理想导线或集总电容 |
| 中等长度线 | 集总电容或集总 RC |
| 长全局金属线 | 分布 RC 或 RC 梯形网络 |
| 高速低阻宽线 | 传输线 / RLC 模型 |
| 电源/地网络 | RC/RLC + IR drop / Ldi/dt 分析 |
| 密集总线 | 线间耦合电容 + 串扰分析 |

## 设计直觉

- 不要一开始就把所有线都建成复杂 RLC 网络。
- 先用手工模型判断主导效应，再决定 SPICE 模型复杂度。
- 长线可以用多段 RC 梯形近似；段数够用即可。
- 对传输线，时间步和反射会显著影响仿真结果。
- 后仿真发现问题后，应回到版图和互连结构本身优化，而不是只调仿真参数。

## 易错点

- 把分布 RC 线误当成一个远端电容，会高估或低估不同节点波形。
- 盲目增加 RC 分段数，可能只增加仿真时间，不提高有效设计判断。
- 忽略线间耦合时，串扰和相邻线相关延迟无法被仿真出来。
- 用无损传输线模型处理强损耗片上线，可能得到过于理想的反射/传播结果。
- SPICE 模型参数单位容易出错，尤其是每米、每微米、每厘米之间的换算。
