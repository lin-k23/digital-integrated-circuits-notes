# 三维集成与 2.5D 集成

## 一句话理解

三维集成（3D integration）试图在垂直方向堆叠多个 active layers，把逻辑、存储、I/O、MEMS 或光电器件放到不同层中，从而缩短互连距离并提高功能密度；2.5D integration 则作为中间方案，通过键合已完成加工的晶圆/芯片并用 vias 连接，降低工艺耦合难度。

## 为什么需要 3D

随着集成度提高，一个主要挑战是：

```text
如何及时把信号送进/送出 computation elements
```

二维平面上继续增加器件密度会让互连成为瓶颈。三维集成的直觉是：

```text
把原本横向很远的模块放到垂直相邻层
```

这样可以减少计算与存储、逻辑与 I/O 之间的距离，降低互连延迟。

## true 3D integration

教材给出的 true 3D integration 思路：

- 在 metal interconnect layers 之间加入额外 active layers。
- 在 bulk CMOS logic processor 上方放置 high-density memory。
- 在顶部 active layer 放置 optical transceivers 或 MEMS/RF interfaces。

示例层次：

```text
T1: Logic
T2: High Density Memory
T3: Optical I/O / MEMS / RF interface
```

## 3D 的潜在优势

### 1. 缩短互连距离

存储器放在逻辑上方，可减少 processor 与 memory 之间的物理距离。

直接收益：

- 降低延迟。
- 降低互连能量。
- 提高带宽密度。

### 2. 异构集成

不同 active layer 可以承载不同类型器件：

- logic。
- memory。
- optical I/O。
- MEMS。
- RF interface。

这样可以把不同电压、性能、材料或 substrate requirement 的器件组合在一个系统中。

### 3. 模糊 chip / package / board 边界

教材强调，未来系统中 chip、substrate、package、board 的边界会逐渐模糊。设计者不能只看 die 内部，而要同时考虑工艺、封装、互连、散热和系统结构。

## 3D 的挑战

### 1. 散热

多层 active devices 堆叠后，内部层的热更难导出。

```text
more active layers -> higher thermal density -> harder heat removal
```

这是教材明确指出的关键问题之一。

### 2. 良率

每多一层 active layer，都可能引入新的缺陷和对准/加工风险。

多层堆叠会让总体良率更难保证。

### 3. 工艺兼容性

不同 active layers 可能要求不同温度、材料、器件结构和工艺步骤。后续层工艺不能破坏前面已完成的器件层。

## 2.5D integration

教材把 2.5D integration 作为 true 3D 到来前的中间方案。

基本做法：

```text
fabricate complete wafers separately
-> bond wafers / chips so circuits overlap
-> etch vias after metallization
-> electrically connect layers
```

优点：

- 每层器件电气性质相似。
- 各 chips / wafers 可分开制造后再键合。
- 避免后续高温工艺破坏已完成电路。

限制：

- 对准精度不足。
- 教材给出 best-case alignment 约 $\pm2\ \mu m$。
- 因此 inter-chip communication 主要受限于 global metal lines。

## 与 MCM / package 的关系

2.5D / 3D integration 和 [[封装互连层级|MCM / advanced packaging]] 之间边界会越来越模糊：

- MCM 在 package/substrate 层面集成多个 die。
- 2.5D 通过更紧密的 wafer / die bonding 与 vias 实现连接。
- true 3D 则进一步把 active layers 嵌入到互连层之间。

它们共同目标都是：

```text
缩短互连、提高密度、降低 I/O 与存储瓶颈
```

## 设计直觉

- 3D 集成的核心收益来自距离缩短，而不是单个 transistor 更快。
- 存储-计算距离缩短对性能和功耗都很关键。
- 越往 3D 发展，热和良率越关键。
- 2.5D 是 true 3D 的折中版本，制造更现实但通信粒度受限。
- 未来系统设计需要同时考虑 chip、package、board 和 cooling。

## 易错点

- 不要把 3D integration 简化为“把芯片叠起来”；true 3D 涉及 active layers 与 interconnect layers 的深度耦合。
- 不要忽略散热，3D 的热问题可能抵消部分性能收益。
- 不要把 2.5D 和 true 3D 混淆；2.5D 对准精度和互连粒度通常更受限。
- 不要以为垂直堆叠总能提升良率；多层工艺会让 yield 管理更困难。
