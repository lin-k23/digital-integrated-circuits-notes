# 时序逻辑章节总览

## 一句话理解

时序逻辑（sequential logic）是在组合逻辑之外加入状态存储的电路：输出不仅取决于当前输入，还取决于过去的状态，因此电路具有“记忆”。

## 组合逻辑 vs 时序逻辑

组合逻辑满足：

```text
当前输出 = 当前输入的布尔函数
```

在组合逻辑中，只要瞬态过程结束，输出与过去历史无关。

时序逻辑满足：

```text
当前输出 = 当前输入 + 当前状态 的函数
```

它通过寄存器、锁存器或其他存储结构保存状态，因此会“记住”过去。

## FSM 高层结构

教材用有限状态机（finite state machine, FSM）作为同步时序系统的基本框图。

典型同步 FSM 包括：

- 组合逻辑：根据当前输入和当前状态生成输出与下一状态。
- 寄存器：保存当前状态。
- 时钟：决定状态何时更新。

状态更新关系：

```text
Current State + Inputs → combinational logic → Next State
Next State 在时钟边沿被寄存器采样 → Current State
```

## 本章关注对象

本章讨论 CMOS 中常用的时序基本单元，包括：

- latch（锁存器）
- register / flip-flop（寄存器 / 触发器）
- oscillator（振荡器）
- pulse generator（脉冲发生器）
- Schmitt trigger（施密特触发器）

本轮优先整理同步寄存器时序约束、存储单元分类和静态锁存器基础。

## 为什么时序单元重要

时序单元决定系统级时钟频率、流水线结构和状态保持能力。

它们影响：

- 最小时钟周期。
- setup / hold 约束。
- 时钟偏斜容忍度。
- 功耗和时钟树负载。
- 数据路径流水化方式。
- 系统对亚稳态和噪声的鲁棒性。

## 设计直觉

- 组合逻辑决定“算什么”，时序逻辑决定“何时保存”。
- 寄存器不是理想采样器，必须满足 setup / hold。
- 寄存器本身会引入 clock-to-Q 延迟和时钟负载。
- 高性能系统中，寄存器开销可能占据相当一部分时钟周期。
- latch-based pipeline 可以提高灵活性，但时钟相位和透明窗口更难管理。

## 易错点

- 不要把 latch 和 register 混用；latch 是电平敏感，register 通常是边沿触发。
- 不要只看组合逻辑延迟，寄存器 $t_{c-q}$ 和 $t_{su}$ 也会限制时钟周期。
- 不要忽略 hold constraint；最小延迟过小也可能导致错误。
- 不要把 static memory 与 dynamic memory 混淆；dynamic 依靠电荷暂存，需要刷新或周期性访问。
