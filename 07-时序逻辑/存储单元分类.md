# 存储单元分类

## 一句话理解

时序电路中的存储单元可以按用途、保持方式和采样方式分类：foreground / background memory 区分是否嵌入逻辑，static / dynamic 区分靠正反馈还是电荷暂存，latch / register 区分电平敏感还是边沿触发。

## foreground vs background memory

### foreground memory

foreground memory 指嵌入逻辑中的小规模状态存储，典型形式是：

- 单个寄存器。
- 寄存器堆中的单元。
- 数据路径流水线寄存器。
- 控制逻辑中的状态位。

第 7 章主要讨论 foreground memory。

### background memory

background memory 指大规模集中式存储阵列，例如：

- SRAM cache。
- 主存储阵列。
- 大容量片上存储器。

它通常通过规则阵列结构获得高密度，并在性能和鲁棒性上做不同折中。

## static vs dynamic memory

### static memory

静态存储依靠正反馈保持状态。只要电源存在，状态就能长期保持。

典型结构：

```text
两个反相器交叉耦合 → 正反馈 → 两个稳定状态
```

适合：

- 可能长时间不更新的寄存器。
- clock gating 后仍需保持状态的模块。
- 配置寄存器。
- 控制状态寄存器。

优点：

- 状态保持可靠。
- 不依赖周期性刷新。

缺点：

- 晶体管数较多。
- 面积和时钟负载可能较大。

### dynamic memory

动态存储依靠 MOS 寄生电容上的临时电荷保存状态。

```text
有电荷 → 代表 1
无电荷 → 代表 0
```

优点：

- 电路简单。
- 速度高。
- 面积和功耗可能较低。

缺点：

- 电荷会泄漏。
- 状态只能保持有限时间。
- 需要刷新或保证周期性访问。
- 对噪声、电荷共享和时钟控制更敏感。

## latch vs register

### latch（锁存器）

latch 是电平敏感（level-sensitive）存储单元。

以 positive latch 为例：

- CLK 高：透明模式（transparent mode），$Q$ 跟随 $D$。
- CLK 低：保持模式（hold mode），$Q$ 保持前一值。

negative latch 相反：CLK 低时透明，CLK 高时保持。

### register / edge-triggered flip-flop（寄存器 / 边沿触发触发器）

register 通常是边沿触发，只在时钟边沿采样输入：

- positive edge-triggered register：在上升沿采样。
- negative edge-triggered register：在下降沿采样。

常见实现方式是 master-slave 结构：

```text
两个互补相位 latch 串联 → 边沿触发寄存器
```

## latch 与 register 的核心区别

| 项目 | latch | register / flip-flop |
|---|---|---|
| 触发方式 | 电平敏感 | 边沿触发 |
| 透明窗口 | 有 | 理想上无 |
| 时序分析 | 可做 time borrowing，但更复杂 | 更直观，常用于同步设计 |
| 常见实现 | transmission-gate latch | master-slave register |
| 设计风险 | 相位、透明窗口、race | setup/hold、clock skew |

## 设计直觉

- 需要长期保持状态时，优先考虑 static storage。
- 高性能数据通路中，dynamic storage 可降低面积和功耗，但要满足周期性刷新和噪声约束。
- latch 更灵活，可做 time borrowing；register 更容易分析。
- 时序单元选择会影响整个 pipeline 的性能、功耗和验证难度。

## 易错点

- latch 不是 flip-flop；latch 是电平敏感，flip-flop / register 通常是边沿触发。
- dynamic storage 不是逻辑功能不稳定，而是状态保持时间有限。
- static memory 不是没有功耗，真实电路仍有泄漏和时钟功耗。
- foreground / background memory 不是按容量绝对大小划分，而是按系统用途和组织方式划分。
