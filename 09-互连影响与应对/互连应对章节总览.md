## 三类互连寄生

### 电容寄生

主要问题：

- capacitive crosstalk
- data-dependent delay
- 大 fanout / 大负载驱动
- bus、clock network、global control wire 的大电容

应对策略：

- 控制线间几何关系。
- 加 shielding。
- 使用 keeper 降低 floating node 风险。
- 使用 differential signaling。
- 使用多级缓冲链驱动大负载。
- 必要时采用 reduced-swing 或 current-mode 技术。

### 电阻寄生

主要问题：

- supply rail 上的 IR drop。
- power/ground network 电压不均匀。
- electromigration。
- 长互连 RC delay。

应对策略：

- power grid / power plane。
- 合理 wire sizing。
- 使用更低电阻材料。
- repeater insertion。
- wire pipelining。
- 从架构层面减少长线依赖。

### 电感寄生

主要问题：

- bonding wire / package pin 引起的 $L\,di/dt$ 噪声。
- ground bounce / supply bounce。
- ringing / overshoot / undershoot。
- transmission line effects。

应对策略：

- 增加 power/ground pins。
- 分离 I/O 和 core supply。
- 控制输出边沿速度。
- board/on-chip decoupling。
- advanced package。
- source / load termination。
- shielding 和明确 return path。

## 本章高频设计场景

- 动态逻辑节点被邻近 full-swing wire 注入噪声。
- bus 上多个邻线同时翻转导致等效电容增大。
- off-chip pad driver 要驱动 10–50 pF 的大电容。
- 多个 I/O 同时切换导致 supply bounce。
- long global wire 的 RC delay 超过一个时钟周期。
- 高速互连表现为 transmission line，需要阻抗匹配。

## 设计直觉

- 互连问题不是“寄生参数很小”的二阶效应，在深亚微米工艺中常常是一阶限制。
- 互连影响同时表现为可靠性问题和性能问题。
- 降低噪声和追求最快速度常常冲突。
- 对大负载，最优速度设计可能带来巨大的面积和能量代价。
- 对长线，单纯换材料只能缓解一两代，架构和布局策略更关键。
- 对高速 I/O，封装和 board 不是外部细节，而是电路模型的一部分。
